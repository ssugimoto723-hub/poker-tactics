## 第1章 プリフロップ戦略

### 1.1 なぜポジションがすべての起点か

後から行動する（IP）ほど、相手の行動という情報を見てから判断できる。この情報量の差が長期EV に直結するため、参加できるハンドの幅はポジションに比例して広がる。

| ポジション | オープン率目安 | 代表的に追加されるハンド |
|-----------|-------------|----------------------|
| UTG | 8〜12% | AA〜JJ, AKs/o, AQs, KQs |
| HJ | 12〜16% | +TT, AJs, KJs, QJs |
| CO | 20〜25% | +99, A9s以上, スーコネ |
| BTN | 30〜40% | +22〜88, 全スーテッド, 弱いオフスーツ |
| SB | 15〜20% | HJに近いタイトさ |
| BB | — | ディスカウントを活かして守備的に広くディフェンド |

**ベット額の基準（6max 100BB）**
- オープン: UTG/HJ = 2bb、CO = 2.3bb、BTN/SB = 2.5〜3bb
- 3ベット: IP から 3倍、SB から 4〜5倍、BB から 5〜6倍
- 4ベット: IP から 2〜2.2倍、OOP から 2.5〜2.8倍

### 1.2 3ベット・4ベットの構成原則

**ポーラー 3ベット（BB の基本形）**
- バリュー側: TT+、AJs+、AQo+ など
- ブラフ側: スーテッドコネクター（67s〜JTs）など低頻度

この二極構成により「3ベットレンジにはナッツもブラフも含まれる」状態を維持し、相手がコール・フォールドどちらを選んでも損をさせる。

**4ベット構成**
- バリュー: QQ+、AKo（議論あり）
- ブラフ: ATs、AJs、KTs 等のスーテッドコネクター（相手の QQ を一枚ブロック、コールされてもエクイティがある）

**リンプを避ける理由**  
プリフロップでリンプすると: ①フォールドエクイティがゼロ ②マルチウェイになり強さが薄まる ③ポジション劣位で参加することが多くなる。

### 1.3 A2s〜A5sの戦略的価値（スーテッドエース）

スーテッドエース（A2s〜A5s）は「ナッツフラッシュドロー＋ホイール（A2345）ストレートドロー」の二重ドロー特性を持つ。

**ブロッカー価値**:
- Aを1枚手に持つことで相手のAKが最大50%減少する
- 3ベットブラフに最適（相手のバリュー3ベットレンジであるAAやAKを一部ブロック）
- コールされても大きいドローエクイティがある

**フロップでの使い方**:
- FD＋ストレートドローが同時に絡んだ場合、フォールドエクイティ＋エクイティの両面でセミブラフが成立
- ナッツFD（最大のフラッシュ）を持つため、逆インプライドオッズのリスクが低い

**変数と傾向変化**:
- スタックが浅いほど（50BB以下）フラッシュ完成時のインプライドオッズが下がるため、3ベットよりフラットコールが有効な場合がある
- マルチウェイではブロッカー価値が薄まるため3ベットの優位性が落ちる

### 1.4 ライト3ベット（ブラフ3ベット）の選択基準

**定義**: バリューの水準（TT+/AQs+）より弱いハンドでの3ベット。フォールドエクイティの確保とレンジバランスの維持を目的とする。

**最適条件**:
1. 相手がスチールしすぎている（BTN開幕で40%+）
2. IP（有利なポジション）からの3ベット
3. 相手がフォールドしやすいタイプ（TAG/Rockなど）

**ハンド選択の原則**:

| 優先 | 理由 |
|------|------|
| 87s/76s（スーテッドコネクター）| バックドアEQあり、コールされても活路がある |
| A2s〜A5s | ブロッカー価値＋ドローEQ |
| KQo | ドミネイトされるリスク（KK/QQ/AKに弱い）→ 避けることが多い |

**2ゾーン構成**:
- バリュー3ベット（TT+、AQs+）: そのままポットを大きくして戦える
- ブラフ3ベット（スーコネ）: フォールドされれば即利益、コールされてもEQある

**注意**: ブラフ3ベットの頻度が過剰になると4ベットで搾取される。相手の4ベット頻度を観察してバランスを保つ。

## 第2章 フロップ戦略（テクスチャ別）

### 2.1 なぜテクスチャで戦略が変わるか

フロップでの CB 判断は「自分のレンジが相手より有利かどうか」で決まる。ボードの構成によって各プレイヤーのレンジがどのくらい「当たるか」が変わるため、同じポジションでも CB 頻度・サイジングは変わる。

### 2.2 テクスチャの分類と CB 方針

#### ドライ + スタティックボード

例: `K♠ 7♦ 2♣`（同スーツなし、連番なし）

- **特徴**: ドローがほぼない。フロップで有利な側が有利なままリバーまで続く。
- **CB 方針**: レイザー（IP 側）は高頻度（80%+）で CB 可能。小サイジング（1/3 ポット）で効率的。
- **なぜか**: コーラー側のレンジにはゴミハンドが多く含まれており、小さい額でも多くを降ろせる。

#### ウェット + ダイナミックボード

例: `J♥ T♠ 9♥`（フラッシュドロー + ストレートドロー豊富）

- **特徴**: ターン・リバーでエクイティが大きく変動する。コーラー側が有利になるケースがある。
- **CB 方針**: 頻度を下げ、打つ場合は大きいサイジング（2/3〜フルポット）でプロテクション。
- **なぜか**: コーラーのドローが多く、安い CB は簡単にコールされてエクイティ実現されてしまう。

#### ペアボード

例: `8♠ 8♥ 2♣`

- **特徴**: ほぼ全フロップで高頻度 CB が発生する（100%近く）。
- **傾向変化**: OOP では若干 CB 頻度が下がる。ペアった数字が高いほど CB 頻度もやや変わる。
- **なぜか**: ペアボードではトリップスを持つ可能性がレイザー側に偏る（タイトなレンジ）。

#### ローカードボード（レイザーが不利なケース）

例: `6♥ 7♠ 8♣`

- **特徴**: BB などの広いレンジには低いコネクテッドカードが多く含まれ、コーラー側がレンジアドバンテージを持つ。
- **CB 方針**: レイザーは慎重。チェックしてフロップを無料で見せるか、大きいサイズで圧力。

### 2.3 CB 判断のフローチャート

```
自分は IP か？
  Yes → レンジアドバンテージがあるか？
           Yes → ドライ: 高頻度小サイズ CB
                ウェット: 中頻度大サイズ CB
           No  → チェックして相手の動きを見る
  No  → レンジアドバンテージがあるか？
           Yes → CB 可（ただし高頻度は避ける）
           No  → 基本チェック。強いハンドのみベット
```

### 2.4 ツートーンボード（Two-tone Board）の考え方

**定義**: 3枚のボードカード中2枚が同スートの状態。全フロップの約55%がツートーン。

**フラッシュ完成確率**:
- ターンで完成: 約19%（残り9アウツ÷47枚）
- リバーで完成（ターン外れ）: 約18%
- ターン＋リバー合計: 約35%

**戦略的影響**:
- サードフラッシュカードが落ちると一気に状況が変わる（レンジが大きく変化する）
- ターンでの保護ベット（プロテクション）が重要になる: フラッシュドローにエクイティを安く実現させない
- CBのサイジングはドライボードより大きめ（2/3ポット以上）が推奨される

**変数と傾向変化**:
- マルチウェイ（3人以上）ではフラッシュドローを持つ相手が複数いる可能性がある → より慎重なサイジングが必要
- モノトーンボード（3枚全て同スート）は別カテゴリで扱う。フラッシュ完成者が多く、ベット頻度が大幅に下がる

### 2.5 レンジCB（Range CB）が有効な条件

**定義**: ほぼ全レンジでフロップをベットする戦略（頻度85〜100%）。バリューとブラフの区別をせず、全ハンドで小サイジングCBを打つ。

**有効条件**（3つが揃うと最も効果的）:
1. **IP（有利なポジション）** からのアクション
2. **ドライ＋スタティックボード** （例: A♠7♦2♣、K♠8♦3♣）
3. **レンジアドバンテージが大きい** （レイザー側がトップカードを大量に含む）

**サイジング**:
- 小さく（1/3ポット）で多くのハンドを降ろすことが目的
- 相手のコールレンジが弱いため、小額でも十分なフォールドエクイティがある

**代表ボード**:
- A♠7♦2♣: BTNがオープン→BBコールの場合、BTNはAKK/KK/AA等を含み大きなレンジアドバンテージ
- K♠8♦3♣: 同様の理由でKハイボードはレイザー有利

**変数と傾向変化**:
- ウェットボードやOOPではレンジCBの有効性が下がる
- BBのコールレンジに低いペアが多いほど（676など）、レンジアドバンテージが逆転する

### 2.6 チェックレンジを強く保つ方法

**なぜ必要か**: フロップチェック＝弱いと読まれると、相手が自由にベットしてターン・リバーでプレッシャーをかけやすくなる。チェックレンジに強いハンドを混ぜることでこれを防ぐ。

**実践的な方法**:
- トップペア以上でも20〜30%の頻度でチェックに回す（GTO的混合戦略）
- 代表例: ブロードウェイボードでのAKフロップ → トップ2ペアの20%をチェック → ターンでトラップ

**効果**:
1. 相手のブラフレイズを防ぐ（チェックが弱いとは限らないと認識させる）
2. フリーカードを得て中程度のハンドをコントロールできる
3. スロープレイとして相手にポットを大きくさせる機会を作る

**変数と傾向変化**:
- ドライなボードほど、チェックレンジに強いハンドを入れる効果が高い（相手のブラフが増えやすい）
- スタックが浅い（50BB以下）場合は混合戦略より明確なバリューベットを優先することが多い

## 第3章 ターン・リバー戦略

### 3.1 ターンの基本特性

ターンは「フロップより静的だがリバーより動的」な位置づけ。主な変化は：

- **SPR（スタック対ポット比）が下がる**: フロップでベットが入った後、ターンからポットが大きくなり、「全部突っ込む」ことへの敷居が下がる。
- **プレイヤーが自分のハンドクラスのインセンティブに従い始める**: 強いハンドはより攻撃的に、弱いハンドはより守りに入る。

### 3.2 ターンでのレンジ二極化

フロップと大きく異なる点：フロップの「広いレンジへの小さい CB」から、ターンは「強いハンドとブラフが大きいベットで二極化」する。

| ハンドの強さ | ターンでの推奨行動 |
|------------|----------------|
| ツーペア・セット・ナッツ | ポットサイズ以上（オーバーベットも）でポット最大化 |
| 強いドロー（フラッシュドロー等）| セミブラフとして大きいベット |
| 中程度（ワンペアなど）| チェックしてポットコントロール・安いショーダウンを狙う |
| 弱いハンド | 大きいブラフか諦める |

**核心の考え方**: 中程度の強さのハンドはチェック。これにより「チェックレンジが強く保たれる」ため、相手も簡単にベットしてこられない。

### 3.3 ターンでバレルを続ける条件

ダブルバレル（ターンでも CB を続ける）には理由が必要。以下のケースで正当化しやすい：

1. **ターンカードが自分のレンジを強化するカード** - 例: 自分のレンジに多い A や K がターンに落ちる
2. **ドローが完成してセミブラフのエクイティが増えた** - フラッシュドローが追加された等
3. **相手がタイトで降りやすい** - フォールドエクイティが高い

逆に以下では続けない：
- ターンカードが相手のレンジを強化（例: 相手 BB のレンジにある 5〜8 がターンに落ちる）
- 中程度の自分のハンドでポットをふくらませても旨味がない

### 3.4 リバーの意思決定

リバーはエクイティが確定する。「バリューか、ブラフか、チェックか」の純粋な判断になる。

**ベットするなら**:
- バリュー: 相手のコールレンジに確実に勝てる
- ブラフ: 相手がフォールドする確率が十分に高い（相手のレンジを観て判断）

**チェックバック（ショーダウン）**:
- 中程度のハンド（コールされると微妙、ベットしても降ろせない）

**ストーリーの一貫性**:  
フロップ → ターン → リバーのベットラインは矛盾なく繋がっていること。リバーで突然ブラフを打っても、フロップからのラインがブラフとして成立していなければ相手に見透かされる。

### 3.5 ディレイCB（Delay CB）

**定義**: フロップでチェックし、ターンでベットする2ストリート戦略。

**使いどころ**:
1. フロップCBに適さないボード（相手レンジが有利、またはウェットボードでCBが薄い）でチェック → ターンで有利なカードが落ちたらベット
2. チェックレンジを強く保ちつつ（2.6参照）、ターンで圧力をかける

**有利なターンカード**:
- 自分のレンジを強化するカード（例: UTGオープン → AやKが落ちる）
- ドローが外れるカード（ツートーンボードでブランクターン）

**利点**:
- ターンは相手も「フロップで降りなかった弱いハンド」を多く持っている → フォールドエクイティが高まる
- フロップCBのプレッシャーなしでターンに進めるため、ポットをコントロールできる
- スロープレイとしての機能も持つ（強いハンドを隠す）

**変数と傾向変化**:
- OOPでは相手のレンジが有利になりやすいフロップが多いため、ディレイCBの機会が増える
- ターンで不利なカードが落ちた場合はチェックを継続する

### 3.6 プローブベット（Probe Bet）

**定義**: OOP（不利なポジション）側が、フロップCBをされずにチェックバックされた後、ターンでベットすること。

**発生状況**:
- IP側がフロップでチェックバック（CBをしない）→ OOP側にアクションが回る → OOP側がベット（プローブ）

**目的**:
1. 相手のチェックバックが示す弱さを突く（「CB打てなかった = 弱い」という仮定）
2. セミブラフ（ドロー持ちで前に出る）

**有効なハンド**:
- ドロー（特にフラッシュドロー・OESD）: エクイティがある状態でフォールドエクイティを加える
- ミドルペア〜ワンペア: 相手がチェックバックした弱いレンジに対して優位を取る

**注意**:
- チェックバック ≠ 必ずしも弱い。IP側がスロープレイ（トラップ）でチェックバックする場合もある
- 相手のプレイスタイルを観察してから使う（常にCBを打つタイプか確認）
- レイズされた場合のフォールドプランを持っておく

**変数と傾向変化**:
- 相手がCBを高頻度で打つタイプ（ダブルバレラー）の場合、チェックバックは比較的弱いシグナル
- 相手がCBを低頻度でしか打たないタイプの場合、チェックバックでも強いハンドが含まれうる

### 3.7 チェックレイズ（Check-Raise）の使い方

**定義**: チェックした後に相手のベットに対してレイズすること。

**2種類のチェックレイズ**:

| 種別 | 使うハンド | 目的 |
|------|-----------|------|
| バリューCR | セット、ナッツFD+ペア、ストレート | 相手を引き込みポットを最大化 |
| ブラフCR | ドロー系（バックドアFD等）、弱いハンド | 相手の狭いバリューレンジをフォールドさせる |

**有効なボード**:
- **ウェット/ダイナミックボード**: 多くのドローが存在し、CR後の継続ラインが説明しやすい
- ドライボードでのCRはバリューが少なく、相手がフォールドしにくい（CR後に矛盾が生じやすい）

**ブラフCRが成立する条件**:
- 相手のベットレンジが強いバリュー側に偏っている（相手がドライボードで小CBしている等）
- OOPで自分のチェックレンジに強いハンドが含まれている（2.6参照）
- 自分のハンドが完成したときに強い（ブランクカードで詰む前提でない）

**注意**:
- CR後に相手がコールしたとき、ターンでどう続けるか（バレル or チェック）を打つ前に決めておく
- ブラフCRのターン継続率が低いと相手にコールされやすくなる（続けないなら打つな）

## 第4章 オッズ・エクイティ

### 4.1 ポットオッズ（基礎）

```
ポットオッズ（%）= コール額 ÷ （現在のポット + コール額）× 100
```

**例**: ポット $100、相手ベット $50  
→ 50 ÷ 150 = **33.3%**  
自分のエクイティが 33.3% 以上 → コール有利

### 4.2 アウツとルール・オブ・2と4

| ドロー | アウツ | フロップでの EQ | ターンでの EQ |
|-------|-------|----------------|-------------|
| フラッシュドロー | 9 | 36% | 18% |
| OESD | 8 | 32% | 16% |
| ガットショット | 4 | 16% | 8% |
| オーバーカード 2枚 | 6 | 24% | 12% |
| FD + OESD | 15 | 60% | 30% |

**ルール・オブ・2と4**:  
- フロップ（2枚残り）: アウツ × 4  
- ターン（1枚残り）: アウツ × 2

### 4.3 エクイティの堅牢性（Robust Equity）

**概念**: EQ の大小だけでなく「相手のアクションに対して価値が減少しにくいか」を評価する。

- **ロバスト EQ**: 相手がレイズしてもコールしても EQ が下がりにくい（例: ナッツドロー、セット）
- **バルネラブル EQ**: 相手の強いアクションで価値が大きく下がる（例: ワンペアがレイズされる）

**ベットサイジングへの連動**（Amu の分類）:

| EQ の大きさ | 堅牢性 | 推奨アクション |
|------------|-------|--------------|
| 大きい | ロバスト | 大きいベット。相手の強いハンドとショーダウンを狙う |
| 小さい | ロバスト | 積極的なベットで相手をフォールドさせ、後の EQ 実現を狙う |
| 大きい | バルネラブル | 相手の弱いハンドをフォールドさせる（プロテクション） |
| 小さい | バルネラブル | チェック。アグレッシブなプレイは避ける |

**代表例**: `7♥ 8♥` を持ちフロップが `5♥ 6♣ K♦`
- ストレートドロー（ロバスト、小 EQ）→ 積極的に CB/レイズで相手をフォールドさせつつ EQ 実現を狙う

### 4.4 インプライドオッズ

現在のポットオッズが不十分でも、ドロー完成後に追加で稼げる額を加味できる。

**使える条件**: 相手のスタックが深い、ドロー完成が相手に気づかれにくい（隠れたドロー）、相手がペイオフしやすいタイプ。

**逆インプライドオッズ**: ドロー完成時に「2番手」になる可能性がある場合は大きく割り引く（例: フラッシュドローで相手がより大きいフラッシュを持ちうる）。

## 第5章 GTOの基本理念

### 5.1 GTO とは何か

GTO（Game Theory Optimal）= 「どんな相手の戦略に対しても非負の EV を保証する、搾取されない戦略」  
**搾取ではなく搾取されないことを目的とした防衛的な基準点**。

ポーカーを非協力ゲーム・ゼロサムゲーム・不完全情報ゲームとして捉えると、GTO は「ジャンケンの 33/33/33%」と同じ——読まれても価値がゼロにならない状態。

**GTO の誤解**: GTO は「読まれても大丈夫」であって「常に最高 EV」ではない。読みのある弱い相手に対しては、GTO から逸脱したエクスプロイト戦略の方が高 EV になる。

### 5.2 混合戦略と最小防御頻度（MDF）

**混合戦略**: 同じハンド・同じ状況でも確率的に行動を使い分ける。  
例: トップセットを 80% ベット / 20% チェック → 「チェック = 弱い」と決めつけられなくなる。

**MDF（最小防御頻度）**:
```
MDF = ポット ÷ （ポット + ベット）
```
例: ポット $100、ベット $50 → MDF = 67%  
67% 以上のレンジでコール or レイズしていれば、相手の純粋ブラフは長期的にプラスにならない。

### 5.3 GTO vs エクスプロイト：使い分けの考え方

| | GTO | エクスプロイト |
|--|-----|--------------|
| 目的 | 搾取されない | 相手の弱点から最大限稼ぐ |
| リスク | 上振れが少ない | 読み間違えると逆搾取される |
| 使いどころ | 読みにくい強敵、オンライン Reg | 明確な癖・リークのあるプレイヤー |

**推奨学習フロー（前元氏の整理）**:
1. フロップテクスチャ別の簡易戦略を覚える（パターン把握）
2. GTO の性質を理解する（レンジアドバンテージ、ナッツアドバンテージ、EQ 堅牢性がベットサイズを決める）
3. ターン・リバーの代表的な簡易戦略を知る
4. ソルバーで自分のスポット・疑問点を検証して感覚を調整する
5. エクスプロイトを学ぶ（リーク・テルの認識 → 対応方針）
6. 頻度管理（ブラフ/バリューハンドの選定基準を決める）

## 第6章 レンジ思考とベットサイジング

### 6.1 レンジで考える（1ハンドでなく集合で）

相手の「1 つのハンド」を当てようとするのではなく、「相手がそのアクションをする可能性のあるハンドの集合（レンジ）」を考える。これが中〜上級への分岐点。

### 6.2 レンジアドバンテージ vs ナットアドバンテージ

- **レンジアドバンテージ**: 特定ボードでの自分のレンジ全体の期待勝率が相手より高い状態。CB 頻度を上げる根拠になる。
- **ナットアドバンテージ**: 特に「最強ハンド（ナッツ）を持っている可能性が高い」こと。ナットアドバンテージがある側は大きいベットが正当化される。

**具体例: A♠ K♦ 7♣ ボード（BTN オープン vs BB コール）**
- BTN のレンジ: AA、KK、AK、AQ 等の強いハンドを含む
- BB のレンジ: より広く、弱いハンドも多い
- → BTN がレンジ + ナットアドバンテージを持つ → 高頻度・小サイズ CB が有効

**具体例: 6♥ 7♠ 8♣ ボード（UTG オープン vs BB コール）**
- BB のレンジ: 56, 79, 45 などの低いコネクテッドカードが豊富
- UTG のレンジ: 高いハンドが多く、このボードでは「外れ」が多い
- → BB がレンジアドバンテージ → UTG はチェックが多くなる

### 6.3 ポラライゼーションとレンジの類別

レンジには3種類ある（Amu の整理）：

| レンジ種別 | 特徴 | よく持つ側 |
|-----------|------|----------|
| **リニア** | 強いハンドから順に集めたレンジ | オリジナルレイザー |
| **キャップド** | ナッツを持ちにくい（上限がある）| コーラー（強いハンドで 3ベットしているため） |
| **ポラライズド** | 強いハンドとブラフのみの両極端 | アグレッサーのベットレンジ |

**ポラライズの核心**: EQ（エクイティ）が等しくても、EV（期待値）に大きな差が生じる。ポラライズしたレンジを構築するとリニアや キャップドと同 EQ でも一方的に高い EV を持てる。

### 6.4 ブラフ頻度の計算

リバーでの最適ブラフ比率はベットサイズで決まる（MDF から逆算）:

| ベットサイズ | 相手のポットオッズ | ブラフ比率 |
|------------|----------------|----------|
| 1/2 ポット | 33% | 約 25%（バリュー 3:ブラフ 1）|
| 等倍 | 50% | 約 33%（バリュー 2:ブラフ 1）|
| 2 倍 | 67% | 約 40%（バリュー 3:ブラフ 2）|

### 6.5 セミブラフの優先基準

優れたセミブラフ = 「今は弱いが EQ がある」ハンドでのベット。

**優先順位**:
1. アウツが多い（FD = 9、OESD = 8）
2. ドロー完成時にナッツになる（相手の 2番手 FD に負けない等）
3. ブロッカーを持つ（相手のナッツレンジを構成するカードを手に持っている）

### 6.6 薄いバリューベット（Thin Value Bet）

**定義**: 強くはないが、相手のコールレンジ全体に対して期待値プラスになるベット。「バリュー vs ブラフ」の中間にある精緻な判断。

**成立条件**:
- 相手のコールレンジに対して50%超で勝てる
- 相手がライトコール傾向（コールレンジが広い）
- ドローが外れてエア（何も完成していないハンド）でコールしてくる場面

**代表例**:
- リバーでフラッシュドローが外れ、ボードが7-8-Q-2-Aになった場面でワンペアをベット → 相手が「ドローを追っていた弱いハンド」でコールしやすい
- ターンでワンペアをベット → 相手のチェックコール幅がトップペア未満のハンドを多く含む

**注意**:
- 相手がタイトな場合、コールレンジが強くなりすぎてバリューにならない
- ベットした後にレイズされるとバリューを逆用される（薄いバリューはチェックコールへの誘導も多い）
- リバーでの薄いバリューはブロッカーの有無でも判断が変わる（相手のナッツをブロックしているか）

**変数と傾向変化**:
- スティーションタイプには大きいサイジングで薄いバリューを連打できる
- タイトアグレッシブな相手には薄いバリューよりショーダウンバリュー狙いのチェックが多くなる

### 6.7 オーバーベット（Overbet）の使いどころ

**定義**: ポットの100%以上のベット。通常はポットサイズの1.5〜3倍の額。

**有効条件**（いずれかが満たされる状況）:
1. **ナッツアドバンテージが一方的にある**: 自分だけがナッツを持ちうる状況。相手のレンジにはナッツが含まれない
2. **相手のレンジがキャップされている**: 相手が強いハンドを持てない状況（例: フラットコーラーはAA/KKを持ちにくい）
3. **ブリックターン/リバー**: ドローが外れて相手のレンジが弱化し、コールしてくるハンドが減る

**バリュー側**:
- ナッツ付近のハンド（ナッツフラッシュ、ナッツストレート）でポットを最大化
- 相手のコールレンジを広げて最高EVを取る（大きいベットに間違ったコールをさせる）

**ブラフ側**:
- 相手がコールしにくいオッズを提示してフォールドさせる
- ナッツブロッカーを持っているハンドを優先（AsFDブロッカー等）

**代表例**:
- K♥Q♥J♥ ボードでAhを持つ場合: ナッツFDを持ち、相手のナッツ（AhKh等）をブロック → オーバーベット有効
- リバーでフラッシュが完成したとき: バリュー側（フラッシュ完成）とブラフ側（FDを持っていたが別のハンドで）の両方でオーバーベット構成を作る

**変数と傾向変化**:
- スタックが浅いほどオーバーベットの幅が限られる（残りスタックが少ない）
- 相手がコールステーションの場合、ブラフ側のオーバーベットは逆効果

## 第7章 エクスプロイト戦略

### 7.1 エクスプロイトの前提：観察 → 確証 → 搾取

ライブポーカーでは多くのプレイヤーが GTO から大きく乖離している。その乖離を利用することが「エクスプロイト」。

**プロセス（松尾氏の実践）**:
1. **観察**: 相手のアクションパターンを観察する（フロップでの CB 頻度、ターンのバレル、ショーダウンのハンド等）
2. **確証**: 「コールしすぎ」「フォールドしすぎ」等の傾向を複数回確認して確証を得る
3. **搾取タイム**: 確証が取れたら継続的に搾取する

### 7.2 代表的なリークと対応方針

**① コールしすぎる（ステーション）**
- 症状: 弱いハンドでも何でもコールする
- 対応: ブラフを減らしてバリューベットを厚くする。サイジングを大きくして EV 最大化。

**② フォールドしすぎる（ブラフカッチャーが少ない）**
- 症状: ベットされると素直に降りる
- 対応: ブラフ頻度を増やす。CB を多打ちする。ターンでのバレルを増やす。

**③ ショーダウンバリューのないハンドでチェック・コール**
- 症状: ドローが外れてもチェック・コールし続ける
- 対応: このタイプには大きいバリューベットが有効（コールレンジが広すぎる）

**④ ビッグハンドをスロープレイしすぎる**
- 症状: セット等をチェックして相手に安いカードを見せ続ける
- 対応: このタイプのチェックは信用できないため、チェックに対しても積極的に動く

### 7.3 フロップ CB とテルの組み合わせ

松尾氏・前元氏の観察から:
- **フロップの CB 頻度**: ポジションとボードが決まればある程度 CB 率が決まる（GTO の簡易戦略）。この率から大きく外れているプレイヤーはリークを持っている。
- **ターンのバレル**: ダブルバレル頻度が著しく低い（ほぼ CB 止まり）→ ターンのチェックは弱いことが多い → ターンでのフロート（コール後にターンで逆ベット）が有効。
- **レンジが弱くなる人間のアクションと動作**: 特定のベットサイズ、考え込む時間、チップの動かし方等がテルになりうる。

### 7.4 頻度管理（ブラフハンドの選定）

ブラフの「どのハンドを選ぶか」は頻度管理の問題。

**考え方**: ダブルバックドアドロー（2 段階のドローが必要なもの）を「全部ブラフに回す」と頻度が過剰になりすぎる。

**実践的アプローチ**:
- 「今日はハートのダブルバックドアがきたらブラフしよう」といったマイルールで頻度をコントロール
- 重要なのは「特定のハンドを暗記する」より「自分の頻度が適正か」を意識すること

### 7.5 MES戦略（Maximally Exploitative Strategy）

**定義**: 相手がGTOから大きく乖離している場合に、こちらもGTOを捨ててEVを最大化する戦略。

**GTOとの違い**:
| | GTO | MES |
|--|-----|-----|
| 目的 | 搾取されない | 相手の弱点を最大限に搾取する |
| 前提 | 相手の戦略に関係なく安全 | 相手のリークが確認できること |
| リスク | 上振れが少ない | 読み間違えると逆搾取される |

**前提条件**:
- 相手のリークが高確率で確認できること（十分なサンプル数）
- サンプル不足でMESを適用すると逆搾取される可能性がある

**代表的なMES適用例**:

| 相手のリーク | MESによる対応 |
|------------|-------------|
| 常に弱いハンドでコールしすぎる | ブラフゼロ、バリューのみで厚くベット |
| フォールドしすぎる（折れすぎ）| バリューなしでも高頻度でベット/レイズ |
| ショーダウンなしのブラフを多用 | コール頻度を大幅に上げる |

**注意**:
- MESは短期的な最大化。相手が気づいて戦略を変えたらGTOに戻す
- ライブポーカーでは複数セッション観察してから適用するほうが安全

### 7.6 スタット読み（VPIP / PFR / 3bet）

オンラインポーカーでは統計ツール（HUD）を使ってスタットを確認できる。ライブでも長期観察で同様の傾向を把握できる。

**主要スタット**:
- **VPIP**（Voluntarily Put money In Pot）: 自発的にチップを入れる頻度（ビッグブラインドの強制除く）
- **PFR**（Pre-Flop Raise）: プリフロップでレイズする頻度
- **3bet%**: 3ベットする頻度

**プロファイル別の目安**:

| プロファイル | VPIP | PFR | 3bet |
|------------|------|-----|------|
| タイトアグレッシブ（TAG）| 15〜20% | 12〜16% | 5〜8% |
| ルーズアグレッシブ（LAG）| 25〜35% | 20〜28% | 8〜12% |
| タイトパッシブ（Rock）| 10〜15% | 5〜8% | 2〜4% |
| ルーズパッシブ（Fish）| 35%+ | 8%以下 | 3%以下 |

**VPIP − PFR の差**:
- 差が10%以上 → コールが多い傾向（コールステーション）
- 差が小さい（3%以下）→ ほぼレイズかフォールドのみ（タイトアグレッシブ）

**活用方法**:
- Fish（VPIP高/PFR低）への対応: ブラフ減らしてバリューを厚くする
- Rock（VPIP/PFR共に低）への対応: ブラフを増やしてフォールドを引き出す
- LAGへの対応: 3ベット頻度が高いため4ベットバリューのレンジを広げる

### 7.7 マルチウェイ（3人以上）のGTO傾向

**理論的背景**: HUポーカーにはナッシュ均衡解が存在するが、3人以上のマルチウェイでは数学的な均衡解の存在が保証されない。実践ではHUの原則から逸脱した調整が必要。

**実践上の指針**:

| 項目 | HU（1対1）| マルチウェイ（3人以上）|
|------|---------|-------------|
| CB頻度 | 高い（50〜80%）| 下げる（30〜50%）|
| ブラフ頻度 | 標準 | 大幅に減らす |
| バリュー基準 | ワンペア以上 | ツーペア以上が推奨 |
| ポジション | 重要 | より重要 |

**ブラフを減らす理由**:
- 複数の相手を全員フォールドさせる確率が積算で低くなる
- 例: 1人を降ろす確率70% → 2人を同時に降ろす確率は49%（0.7×0.7）

**バリューレンジを絞る理由**:
- マルチウェイではどれかの相手がヒットしている確率が高い
- ワンペア程度ではナッツとの対決で不利

**チェック頻度を上げる理由**:
- ドローや弱いハンドをチェックしてEQを守る
- 複数のプレイヤーがいるためポットコントロールの価値が上がる
- ベットにより多くのプレイヤーが絡む → ポットオッズが変わる

**変数と傾向変化**:
- スリーウェイ（3人）とフォーウェイ（4人）で戦略の調整幅が変わる
- ポジションが最も重要: BTNは最後のアクションでマルチウェイでも有利

## 第8章 トーナメント固有の調整

> **注意**: 多くの GTO コンテンツはキャッシュゲーム（100BB）を基準にしている。トーナメント（MTT）は別のゲームとして差分を把握することが重要。

### 8.1 アンティによるプリフロップ変化

**MTT の最大の違い**: アンティの存在が BB のディフェンス EV を変える。

- アンティがある分、ポットが膨らんでいる → BB のコールオッズが改善される
- 結果: **BB のディフェンスレンジが広がる**
- 影響: IP（レイザー側）は MTT の方がキャッシュより高頻度で CB を打てる

**ボード別の主な差異**（キャッシュ 100BB vs MTT 80BB アンティあり）:
- **A ハイボード**: BB に「ゴミレンジ」が増えるため、IP は安い CB が有効
- **ローカード連番ボード**: BB コールレンジが広がりトリップスコンボが増えるため、IP の有利性がやや減少
- おおむね **9 割のフロップで MTT の方が CB 頻度が高くなる**

### 8.2 ICM と cEV・$EV の違い

| 概念 | 定義 | 使う場面 |
|------|------|---------|
| cEV（チップ EV）| チップの増減だけを評価 | キャッシュゲーム / 深いスタックのトナメ序盤 |
| $EV（ドル EV）| 賞金への影響を考慮した評価 | バブル / ファイナルテーブル近辺 |

**ICM の核心**: チップには逓減効果がある。チップを 2 倍にしても $EV は 2 倍にならない。負けてチップを全部失えば $EV はゼロ。→ 「獲得の利益 < 損失のコスト」という非対称性が生まれる。

### 8.3 バブル戦術

| スタック | ICM プレッシャー | 方針 |
|---------|---------------|------|
| ビッグ | 低い（最も自由） | アグレッシブにミドル・ショートを攻める |
| ミドル | 最も高い | フォールドを増やす。薄いコールは避ける |
| ショート | 比較的低い | プッシュ/フォールドで強気に動く |

**典型的な ICM 自爆**:
- cEV では有利なコールでも、アウトになるリスクの $EV コストが大きい場合にフォールドが正解
- 同格スタックとのコンフロンテーション（両者に $EV コストが発生し、ビッグスタックが得をする）

### 8.4 ファイナルテーブル（FT）戦略

1. 自分より短いスタックが先にバストアウトするよう誘導する（賞金順位が上がる）
2. 同等スタックとの直接対決は避ける
3. ビッグスタックからのプッシュへのコールは非常に厳しいレンジで

### 8.5 スタック深度（BB 数）による戦略変化

トーナメントではスタックが浅くなるにつれてプリフロップレンジが変化する。前元氏の整理:

- トーナメントでは基本的に「一段階ポジションを緩くした時のレンジ構成」に近づく
- スーテッドコネクターやギャッパーの扱いはキャッシュと本質的に変わる
- 詳細な MTT GTO 記事は少ないため、キャッシュ GTO を基準としてターン・リバーの変化性質を追う形が現実的
